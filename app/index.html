VASSET

LG, TINNO, ZTE 단말 제조사 대상으로 Nextreaming의 동영상 편집 Editor SDK가 판매되어 해당 SDK에사 사용 할 수 있는 편집 아이템 요소를 구매 할 수 있는 스토어 앱이다.

Google PlayStore에는 LGE와 TINNO 에서 사용 할 수 있는 앱이 등록되어있다.


개발자 단독으로 개발을 완료한 앱이며, 기본 UI에서 부터 Download와 Install Service, 고객사 Editor 간의 IPC 통신 (AIDL)까지 모두 개발하였다.


독립적으로 실행되지 않는 Hidden App이기 때문에 Launcher App에 노출되지 않으며, 반드시 LGE의 Video Editor 혹은 TINNO의 Video Editor에 의해 실행되어야 한다.


Android Databinding이 익숙하지 않아 MVC,MVP 패턴을 이용해 디자인 되어있다.

This is as Store Application for Nexstreaming's Editor SDK Customers LG, TINNO, ZTE. Users can download assets that is used editing by thier Editor Application

Now published LGE and TINNO to Google PlayStore.


This application was developed only one Developer. I made all of module Download to Install service even ICP (AIDL between Editor SDK)


It is not stand alone application. So it does not showing on Launcher App. And This Application must be launched Video editor of LGE or TINNO.


I am not used to Android Databinding. So used MVC,MVP pattern.

MEMEBOX

MEMEBOX에서 개발 관리되고있는 B2C Commerce앱이다.

Google PlayStore에는 미미박스로 현재 등록되어있다.


미미박스가 개발이 완료되고 새로운 기능들을 추가 할 시점에 개발에 참여하였다. Android OS 5.0(Lollipop)에 사용 할 수 있는 UI Touch feedback과 Animation, RestAPI를 통한 페이지 뷰 개발, Custom View 개발 및 UI 성능 개선이 주 업무였다.


Network reponse 처리가 Java Annotation 을 이용하여 Observer pattern으로 다중으로 처리되고 있었는데, Android App 특성상 비효율적 이기 때문에 많은 UI 퍼포먼스 개선을 후 진행을 시도하려고 했다. 하지만 단조로운 작업에 대한 개인적인 갈등으로 오랜기간 프로젝트에 참여하지 않아 시도하지 못 하였다.

MEMEBOX is B2C Commerce application.

MEMEBOX was published to Google PlayStore.


When MEMEBOX was almost done and planing new features, I was involed to MEMEBOX team. Then my main job was improved UIs and functions. So I applied Android OS 5.0(Lollipop) UI Touch feedback과 Animation, developing Page view by RestAPI, Custom View.


All of Network response were procced though multiple Java Annotation as Observer pattern. It was inefficient in Android App. So I was planing to improve that after improving UI performance. But I did not the task because leave MEMEBOX team.
